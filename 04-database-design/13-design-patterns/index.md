# 13 — Design Patterns

Proven, recurring architectural solutions for common database scenarios. More valuable to an AI coding agent than isolated best practices, because the agent can recognize a use case (SaaS multi-tenancy, event sourcing, CQRS) and apply a complete, production-tested pattern rather than improvising. Every file follows the 12-section research template.

## Research template (every file follows this structure)

1. Definition & core concepts
2. Why it exists / what problem it solves
3. What breaks in production without it
4. Best practices
5. Common mistakes / anti-patterns
6. Security considerations
7. Performance considerations
8. Scalability considerations
9. How major companies implement it
10. Decision checklist (when to use / when to skip)
11. AI coding-agent implementation guidelines
12. Reusable checklist
