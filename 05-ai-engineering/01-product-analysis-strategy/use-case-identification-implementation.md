# Use-Case Identification

## 1. Definition & Core Concepts
Use-Case Identification is the process of mapping specific business problems to appropriate AI architectural patterns (e.g. RAG, agentic workflows, text classification, semantic search) based on data characteristics, user requirements, and constraints.

## 2. Why It Exists / What Problem It Solves
It prevents the "hammer looking for a nail" anti-pattern. Identifying the exact nature of the AI use case helps developers choose the simplest model class, prompt pattern, and database architecture.

## 3. What Breaks in Production Without It
- **Over-engineered Architectures:** Deploying complex autonomous multi-agent loops to solve a simple text-extraction task, inflating latency and cost.
- **Incompatible Tech Stacks:** Selecting a standard vector database when the target search use-case requires exact keyword matching (missing lexical features).
- **Out of Memory crashes:** Feeding massive customer documents into direct prompts when the use-case requires a RAG pipeline.

## 4. Best Practices
- **Map problems to patterns:**
  - *If* search across documents, *then* use **RAG / Vector Database**.
  - *If* data extraction or formatting, *then* use **Structured Output (JSON schema)**.
  - *If* autonomous actions execution, *then* use **Function Calling / Agents**.
- **Quantify input size constraints:** Match document sizes to prompt limits.

## 5. Common Mistakes / Anti-Patterns
- **Agentic loops for static reports:** Running heavy, recursive agent reasoning steps to compile standard tables.
- **Direct Prompting for Knowledge Retrieval:** Injecting entire company manuals into a prompt to answer user questions, exhausting context limits.

## 6. Security Considerations
- **Boundary Controls:** Identify if the use case requires sandboxed execution environments (e.g., executing code generated by agents).

## 7. Performance Considerations
- **Caching Suitability:** Determine if the use-case requests are repetitive (allowing prompt caching) or highly dynamic.

## 8. Scalability Considerations
- **Concurrency Scaling:** Identify if use-case throughput spikes require horizontal queue scaling.

## 9. How Major Companies Implement It
- **GitLab:** Splits AI features into isolated use cases (Duo Chat utilizes RAG; Code Suggestion runs low-latency code models), optimizing models per task.
- **HubSpot:** Implements isolated prompt workflows for database updates while restricting agents to task execution.

## 10. Decision Checklist (Use-Case Mapping Matrix)
- Use **Simple RAG** when:
  - Task is answering questions from private document libraries.
- Use **Structured Extraction** when:
  - Task is converting PDF invoices to JSON tables.
- Use **Autonomous Agents** when:
  - Task requires executing multi-step APIs, deciding options, and correcting errors.

## 11. AI Coding-Agent Implementation Guidelines
- Document the target AI pattern (RAG, Extraction, Agent) in the system design before writing prompt templates or database schemas.

## 12. Reusable Checklist
- [ ] Business problem mapped to a specific AI pattern (RAG, Extraction, Agent)
- [ ] Model constraints matched to the use-case complexity
- [ ] Sandboxed runtimes configured for code execution use-cases
- [ ] Database requirements (vector vs relational) defined for the search task
- [ ] Prompt templates isolated by use-case domains
- [ ] Token usage logging configured per feature
