# AI Customer Support Template

## 1. Definition & Core Concepts
An AI Customer Support system is a template designed to handle customer inquiries, retrieve account context, search company knowledge bases, invoke safe transactional tools, and escalate issues to human support staff.

## 2. Why It Exists / What Problem It Solves
Customer support features must be accurate, secure, and fast. An enterprise customer support template orchestrates user session verification, strict RAG retrieval of corporate policies, tool-calling execution parameters, and human-in-the-loop escalation paths to ensure safe, compliant assistance.

## 3. What Breaks in Production Without It
- **False Commitments:** The bot hallucinates a policy and commits to sending a refund or discount the company does not support.
- **Data Leaks:** The chatbot displays account data belonging to tenant A when queried by tenant B.
- **Stalling on Outages:** The bot fails to resolve a client issue during system outages and has no mechanism to escalate the issue or create a human support ticket.

## 4. Best Practices
- **Model Selection:** Use fast models (e.g. GPT-4o-mini, Claude 3.5 Haiku) for conversational responses, and high-reasoning models for tool selection routing.
- **Context/Prompt/Knowledge Strategy:** Ground responses strictly on retrieved knowledge articles (e.g. "Only answer using the context. If the context does not contain the answer, say 'Let me transfer you to a human agent'").
- **Agent/RAG Pattern:** Use a RAG framework with a Router Agent that decides if queries require knowledge search, account lookup tools, or human ticket escalation.
- **Evaluation:** Run automated judges checking for grounding, brand tone compliance, and safety blocks.
- **Deployment:** Deploy chat endpoints alongside ticketing system queues (like Zendesk, Jira Service Desk) to handle handoffs.

## 5. Common Mistakes / Anti-Patterns
- **Allowing autonomous writes:** Letting the model edit account balances, change shipping addresses, or issue refunds without human-in-the-loop review.
- **Generic fallback messages:** Informing the customer "I think the policy is X but I'm not sure," creating corporate liability.

## 6. Security Considerations
- **Boundary Isolation (Multi-tenant checks):** Ensure that the database filters session lookups using verified tenant IDs passed by application session tokens, not parameters generated by the model.

## 7. Performance Considerations
- **Knowledge Caching:** Cache popular help documentation articles in memory to reduce vector database query times.

## 8. Scalability Considerations
- **Handoff queue management:** Ensure that when a human handoff occurs, the conversation transcript is structured, summarized, and routed cleanly to help desk interfaces.

## 9. How Major Companies Implement It
- **Klarna:** Operates AI customer assistants that handle millions of support queries, utilizing strict RAG and ticketing handoffs to resolve support workloads while matching human customer satisfaction scores.

## 10. Decision Checklist (Support Bot Blueprint)
- **Model Selection:** GPT-4o-mini / Haiku (Conversation) -> Claude 3.5 Sonnet (Tool Router).
- **Grounding Strategy:** Strict grounding prompt with fallback to human transfer.
- **HITL Integration:** All write tools (refund, cancel order) require human agent approval before execution.
- **Escalation Path:** Transfer to Zendesk agent chat interface or create support ticket.

## 11. AI Coding-Agent Guidelines
- Write API endpoints that validate tenant authorization, query knowledge bases, execute read tools, and route unresolved queries to ticketing systems.

## 12. Reusable Checklist
- [ ] Model system prompt enforces strict grounding and bans extrapolation
- [ ] Router agent triages requests between search, account tools, and handoffs
- [ ] Safe read-only database tools isolate user accounts via tenant tokens
- [ ] Human escalation pathway configured for unresolved or high-risk topics
- [ ] Output moderation filters block toxic or off-tone generations
- [ ] Handoff workflow sends summarized conversation transcripts to help desks
