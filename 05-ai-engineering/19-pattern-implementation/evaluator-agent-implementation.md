# Evaluator Agents

## 1. Definition & Core Concepts
An Evaluator Agent (often called a Judge Agent) is a specialized AI system configured to assess, grade, and audit the output of other AI models, agents, or software systems against a structured set of guidelines, rules, or rubrics.

## 2. Why It Exists / What Problem It Solves
Evaluating natural language or open-ended code outputs is difficult to automate with traditional software (such as unit tests or regex). Evaluator agents solve this by using advanced semantic reasoning models to act as automated judges, grading outputs on quality, correctness, groundedness, and compliance at production scale.

## 3. What Breaks in Production Without It
- **Silent Output Drift:** A model's outputs become low-quality or off-tone, and developers cannot identify this because standard system logs show successful API calls.
- **Undetected Bias or Hallucination:** Systems serve incorrect facts or biased responses directly to users without retrospective evaluation audits.

## 4. Best Practices
- **Provide Explicit Grading Rubrics:** Structure evaluator prompts with numbered, objective criteria (e.g. "Score 1-5 based on faithfulness to the context").
- **Require Justification Output:** Instruct the evaluator agent to write its reasoning step-by-step *before* generating its final numerical grade, increasing scoring reliability.
- **Calibrate Against Human Annotators:** Periodically compare the scores generated by the evaluator agent with human reviews to measure agreement (e.g., Cohen's Kappa score).

## 5. Common Mistakes / Anti-Patterns
- **Self-Evaluating:** Asking the same model that generated an output to grade its own work within the same session context, leading to self-serving bias.
- **Using Weak Models as Judges:** Deploying small, distilled models to evaluate complex reasoning tasks, resulting in erratic, unreliable evaluations.

## 6. Security Considerations
- **Evaluator Hijacking:** Ensure that the user query or model response being evaluated does not contain prompt injections that trick the evaluator model into signing off on bad data.

## 7. Performance Considerations
- **Asynchronous Execution:** Run evaluation tasks out-of-band on message queues to ensure they do not add latency to the user-facing request path.

## 8. Scalability Considerations
- **Sampling Strategies:** Evaluating 100% of queries on large models is cost-prohibitive. Implement dynamic sampling (e.g. 5% of successes, 100% of user-reported errors) to keep costs manageable.

## 9. How Major Companies Implement It
- **Microsoft:** Uses Azure AI evaluation agents to systematically run side-by-side tests of newly deployed copilot prompts against golden datasets, assessing safety and groundedness metrics.

## 10. Decision Checklist (Evaluator Deployment)
- Use **Automated Evaluator Agents** when:
  - You need to grade open-ended outputs, RAG groundedness, text summarization quality, or safety compliance across thousands of daily requests.
- Use **Traditional Assertions/Linting** when:
  - Outputs are structured formats (JSON, XML, CSV) or compiled code that can be validated deterministically.

## 11. AI Coding-Agent Guidelines
- Implement evaluation wrappers that capture prompt-response pairs, run evaluation prompts on dedicated judge models, and save results in observability repositories.

## 12. Reusable Checklist
- [ ] Evaluator agent is a separate model instance from the generator model
- [ ] Prompt template contains an objective, structured grading rubric
- [ ] Model outputs reasoning step-by-step before generating the grade
- [ ] Evaluation tasks run asynchronously on background worker queues
- [ ] Traffic sampling configured to optimize evaluation costs
- [ ] Evaluator accuracy calibrated against human-annotated test datasets
