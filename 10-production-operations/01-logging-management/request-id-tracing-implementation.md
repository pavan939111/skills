# Request ID

## 1. Backend Application Context
A Request ID is a unique identifier generated by the backend API middleware (or API Gateway) for every incoming HTTP request, enabling engineers to isolate logs related to a single request lifecycle.

## 2. Backend-Specific Pitfalls
- **Failing to return Request IDs in error payloads:** Omitting the Request ID from HTTP 500 responses, leaving users unable to provide a debugging reference to operations support teams.

## 3. Code-Shape Example
```python
# Middleware injecting Request ID to request logs
@app.middleware("http")
async def request_id_middleware(request: Request, call_next):
    request_id = request.headers.get("X-Request-ID") or generate_uuid()
    # Save request ID in logging context
    logging_context.set("request_id", request_id)
    
    response = await call_next(request)
    # Set header to allow client tracing
    response.headers["X-Request-ID"] = request_id
    return response
```
