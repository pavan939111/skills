# 20 — System Design Templates

Reusable, worked end-to-end blueprints for common product shapes — the capstone folder that applies the entire framework (folders 01–19) to a real product, so an agent asked to "build something like X" can go straight to a concrete starting point instead of re-deriving the architecture from scratch. Template: **A — Analysis Template**, extended: each file walks through requirement analysis → capacity planning → architecture selection → the relevant technology strategy decisions → trade-offs → patterns used, for that specific product type.
